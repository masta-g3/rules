Technical Overview

Agent Rules: workflow prompts for AI coding agents

This repository packages reusable skills, subagents, runtime extensions, and workflow rules that make agent-led software development more reliable, lean, and repeatable.

prompt/workflow rules repo human-gated workflow multi-harness sync

Concept

A small operating system for agent-assisted development

The repo is not an app runtime or SDK; it is a set of instructions and deterministic helpers that keep agent work explicit, reviewable, and durable across sessions.

Why it exists

AI coding sessions drift unless work selection, planning, implementation, review, reflection, and commit boundaries are named and user-gated.

What success looks like

Agents maintain real projects with less bloat, clearer handoffs, sparse workflow artifacts, and durable documentation that describes current truth.

Workflow

Skill chain

The core loop makes human decisions visible: choose work, prepare context, plan, implement, inspect, update durable knowledge, then commit cleanly.

Workflow modes

Standalone work and tracked features use the same checkpoints

Standalone

Ad-hoc task

prime → plan-md → execute → review → reflect → commit

Useful for quick work without backlog state. The user invokes each checkpoint directly, and plan names are descriptive, such as DARK_MODE.md.

Feature-driven

Multi-session tracked task

next-feature → prime → plan-md → execute → review → reflect → commit

next-feature chooses ready backlog work; features.yaml tracks status: pending → in_progress → done. Plan names carry ticket IDs, such as auth-001.md.

The difference is state management, not engineering quality: both modes still require explicit review and reflection before commit.

Anatomy

Main directories and how they fit together

skills/

Canonical reusable workflows. Each SKILL.md defines agent behavior; skills/_lib/ holds deterministic shell/Python helpers for shared state operations.

agent-work/

Agent-produced workflow state: backlog in features.yaml, active plans in plans/, archived plans in history/, and sparse evidence in tickets/.

agents/, pi/, extensions/

Reviewers and runtime overlays. agents/ provides focused critics; pi/ and extensions/ make workflow skills visible and ergonomic inside Pi.

Skill map

Skills cluster by workflow responsibility

  1. 01Start and select work
    project-init, epic-init, ticket-init, and next-feature create or choose tracked work without guessing scope.
  2. 02Prepare, plan, and implement
    prime, plan-md, and execute turn selected work into focused edits with baseline verification.
  3. 03Inspect, preserve, and finish
    review, reflect, and commit inspect completed work, update durable docs, archive plans, and finalize tracked state.
  4. 04Support and opt-in automation
    docs-health, test-coverage, workflow-migrate, workflow-orchestrator, and explain-html handle specialized analysis or explicit automation.

Boundaries

Durable knowledge and workflow artifacts stay separate

CONTEXT.md
Project purpose, target user, operating assumptions, and shared terminology.
AGENTS.md
Behavioral rules for coding agents: communication, workflow discipline, testing, and implementation principles.
docs/STRUCTURE.md
Architecture and onboarding guide: directory layout, key files, concepts, and run/build notes.
README.md
User-facing usage reference for workflows, skills, CLI tools, setup, and schema.
agent-work/
Non-durable workflow state and evidence; useful for coordination, not a replacement for durable docs.

Summary

Takeaways for team discussion

  • The repo packages an opinionated but lean workflow for AI coding agents, not product code.
  • The default workflow is user-gated: skills emit handoff labels, and humans or parent agents decide when to advance.
  • State has clear homes: durable truth in docs, tracked work in agent-work/features.yaml, and sparse evidence under agent-work/tickets/.
  • sync-prompts.sh is the distribution bridge: it copies shared skills/subagents to Claude, Cursor, and Pi, overlays Pi runtime assets, and keeps Codex free of tracked-work prompts.
