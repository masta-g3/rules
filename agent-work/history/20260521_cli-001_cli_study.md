**Feature:** cli-001 → User can assess helper and pv/fv CLI usability issues with targeted subagent user-like tests

## Summary

Completed an assessment-only CLI usability study for:

- `skills/_lib/features_yaml.sh` / `features_yaml.py`
- `bin/pv` / `bin/fv`

No utility code was changed. The work produced ticket-local evidence, subagent transcripts, and a prioritized findings report under `agent-work/tickets/cli-001/`.

## Assessment Method

Used deterministic baseline commands plus four targeted subagent user-like tests:

1. Fresh workflow agent — discover next feature, attach a plan file, verify feature state.
2. Error recovery agent — run likely mistakes and evaluate whether stderr gives a correct recovery path.
3. Pipeline automation agent — compose helper output in shell, test stdin JSON and retry/no-op behavior.
4. pv/fv human-like user — inspect TUI help, non-TTY rendering, invalid path behavior, and helper-vs-TUI clarity.

All mutable test work used temp fixture projects under `agent-work/tickets/cli-001/tmp/`; the repo backlog was not used as test data.

## Key Findings

High/medium priority follow-up candidates:

1. Add `features_yaml.sh get/show FEATURE_ID --output text|json` so agents can verify arbitrary feature state without parsing YAML or misusing `describe`.
2. Improve `describe <bad>` and related errors with valid command names and concrete examples.
3. Add helper `--help` examples and document supported update JSON fields.
4. Support `--json -` stdin payloads for `create` and `update` to reduce fragile shell quoting.
5. Make repeated no-op `update` emit `changed:false` instead of always reporting `changed:true`.
6. Clarify `pv/fv` help: humans use TUI tools; agents/scripts should use `features_yaml.sh` for deterministic machine-readable operations.

Lower-priority candidates:

- Improve `complete` recovery hints by including required `--plan-file` in related error text.
- Consider `create --if-missing` only if retry-safe creation becomes a real workflow need.
- Make `pv/fv` invalid-path errors include examples such as `pv .`, `pv agent-work/features.yaml`, and `fv` from project root.
