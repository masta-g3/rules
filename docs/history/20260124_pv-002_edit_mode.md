**Feature:** pv-002 → Edit Mode - Add/Delete/Modify Features

## Overview

Inline editing for the pv TUI. Users can modify `features.json` directly from the terminal: change status, edit fields, add/delete features.

## Key Bindings

| View | Key | Action |
|------|-----|--------|
| Epic | `n` | Create new feature, enter edit mode |
| Feature | `e` | Enter edit mode |
| Feature | `D` | Delete with confirmation |
| Feature | `w` | Save changes (when dirty, in navigate mode) |
| Edit mode | `Tab` | Commit field, advance to next |
| Edit mode | `j/k` | Cycle status options |
| Edit mode | `Esc` | Commit field, exit to navigate mode |
| Any | `q` | Quit (prompts if unsaved changes) |

## Editable Fields

`title`, `status`, `priority`, `description`, `depends_on`, `notes`

## Design Decisions

1. **Edit scope**: Feature view only—keeps complexity localized
2. **Modal editing**: vim-style navigate/edit modes
3. **Pending changes**: Edits buffered until explicit save (`w`)
4. **Dirty tracking**: `[*]` indicator, quit confirmation
5. **ID generation**: `{epic}-{NNN}` pattern with collision avoidance

## Data Model

```python
@dataclass
class EditState:
    mode: str = 'navigate'  # 'navigate' | 'edit'
    field_idx: int = 0
    editing_value: str = ''
    pending_changes: dict = field(default_factory=dict)
    confirm_action: str | None = None  # 'quit' | 'delete'

# Added to State:
edit: EditState | None = None
dirty: bool = False
```
